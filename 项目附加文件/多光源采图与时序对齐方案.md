# 水样袋多光源采图与时序对齐方案

## 1. 背景与目标

水样袋缺陷检测的主要难点不只在模型，而在成像本身：

- 袋体为白色半透明材料，整体对比度低。
- 目标缺陷可能只有 0.1 mm 左右，例如针孔、细小异物、头发丝。
- 白色折痕、压痕、封边纹理和真实缺陷在普通正面光下非常相似。
- 单张 RGB 图像很难同时兼顾透明异物、表面折痕、毛发、针孔和局部污染。

因此建议将原有单光源拍照升级为：

```text
presence 快速检测
-> 抓拍触发
-> 多光源 burst 采图
-> 每张图带高精度时间戳和光源 ID
-> 多图进入缺陷检测队列
-> 多光源结果融合
-> PLC 末端分拣
```

核心目标：

- presence 模型只负责快速判断视野中是否有水样袋。
- 多光源采图负责提升微小缺陷可见性。
- 细致缺陷检测进入异步队列，不阻塞机构动作。
- 相机曝光、PLC 光源切换、图像落盘三者通过统一时间戳和采图 session 关联。

## 2. 分辨率与成像前提

0.1 mm 缺陷如果只占 1 到 2 个像素，检测模型很难稳定识别。建议让目标缺陷至少占 5 个像素，更理想是 8 到 10 个像素。

计算公式：

```text
物面分辨率 = 缺陷尺寸 / 目标像素数
```

以 0.1 mm 缺陷、希望占 5 px 为例：

```text
0.1 mm / 5 px = 0.02 mm/px = 20 um/px
```

如果相机横向为 4096 px，则建议横向视野不超过：

```text
4096 * 0.02 mm = 81.92 mm
```

结论：

- 如果当前一张图拍完整袋体，视野宽度超过 150 mm，则 0.1 mm 缺陷可能只有 2 到 3 px，检测会非常不稳定。
- 如果袋体较大，建议检测关键 ROI，或者采用更高分辨率相机、更高倍率镜头、分区拍摄或线扫方案。
- 光源优化只能提升对比度，不能弥补缺陷像素数严重不足的问题。

## 3. 推荐光源组合

建议将每一面水样袋采集 4 张检测图像，对应 4 种光源状态。

### L0: presence 快速检测光

用途：

- 常亮或低频触发。
- 只用于判断当前视野中是否有水样袋。
- 不追求微小缺陷可见性，只要求袋体轮廓稳定、推理速度快。

建议形式：

- 柔和面光或漫射光。
- 低曝光、低增益。
- 可配合小模型或传统阈值/轮廓方法实现快速 gate。

输出：

```text
bag_present = true / false
presence_confidence
presence_timestamp
```

L0 不进入细致缺陷检测队列，只作为抓拍触发条件。

### L1: 背光 / 透射光

用途：

- 检测头发丝、异物、针孔、局部厚薄变化、透明材料内部杂质。
- 对半透明袋体尤其重要。

成像特点：

- 缺陷表现为透光差异或暗色细线。
- 对头发丝、黑点、微小异物效果通常优于正面光。

适合检测：

- 细小头发丝。
- 黑点、异物。
- 针孔或透光异常。
- 局部材料厚薄不均。

### L2: 低角度暗场光 A

用途：

- 检测表面折痕、压痕、划痕、凸起边缘。
- 让低高度表面结构产生阴影或高亮边缘。

建议形式：

- 低角度条形光。
- 方向建议沿袋体运动方向或与主要折痕方向成夹角。

适合检测：

- 白色折痕。
- 压痕。
- 细小表面划痕。
- 表面附着异物。

### L3: 低角度暗场光 B

用途：

- 与 L2 方向相反或近似正交。
- 解决折痕/发丝方向性问题。

为什么需要第二个暗场方向：

- 细长缺陷对光照方向高度敏感。
- 单方向暗场可能看不到与光线方向平行的头发丝或折痕。
- 双方向暗场可以降低方向盲区。

适合检测：

- 任意方向细线。
- 方向不固定的折痕。
- 贴附在袋体表面的细小毛发。

### L4: 交叉偏振漫射光

用途：

- 抑制塑料表面反光。
- 提升白色袋体上的淡色污染、浅色折痕、表面雾状异常可见性。

建议形式：

- 漫射光或同轴光加偏振片。
- 镜头前加分析偏振片。
- 光源和镜头偏振方向交叉，用于压制镜面反射。

适合检测：

- 淡色污染。
- 反光导致的伪缺陷区分。
- 白色折痕与材质纹理观察。

## 4. 最终建议采图数量

### 标准模式：每面 4 张缺陷检测图

推荐每一面采集：

```text
L1 背光图
L2 暗场 A 图
L3 暗场 B 图
L4 交叉偏振漫射图
```

也就是：

```text
上相机面：4 张
下相机面：4 张
每个水样袋合计：8 张缺陷检测图
```

选择 4 张的原因：

- 背光负责透射类缺陷。
- 双暗场负责方向敏感的表面细线和折痕。
- 偏振漫射负责反光抑制和浅色表面异常。
- 四类图像互补，适合 0.1 mm 级低对比度缺陷。

### 快速模式：每面 3 张图

如果产线节拍压力较大，可先采用：

```text
L1 背光图
L2 暗场 A 图
L3 暗场 B 图
```

即每个水样袋 6 张缺陷检测图。

舍弃 L4 的代价：

- 对淡色污染、塑料反光、浅色折痕区分能力下降。
- 误检可能增加。

### 复检模式：每面 5 到 6 张图

如果某些样本在标准模式下结果不确定，可增加：

- 不同曝光的背光图，用于 HDR。
- 另一组偏振角度图。
- 更低角度的局部暗场图。

复检模式不建议作为默认流程，否则会影响节拍。

## 5. 推荐采图流程

### 5.1 单个相机工位流程

每个相机工位默认下拨杆处于挡停状态，水样袋到位后进入视野。

```text
1. L0 presence 光保持可用
2. presence 模型持续快速检测是否有袋
3. 检测到 bag_present=true
4. 生成 capture_session_id
5. 锁定当前工位 bag_id / station_id / side_id
6. 执行多光源 burst 采图
7. 采图完成后执行 PLC 拨杆动作
   - 下拨杆 release
   - 上拨杆 push
   - 上拨杆 restore
   - 下拨杆 restore blocking
8. 多光源图像组进入 defect_queue
```

注意：

- 缺陷检测不应阻塞拨杆动作。
- 抓拍完成后即可放行，让水样袋进入下一拍摄位置。
- defect_queue 可以慢一些，只要末端分拣前完成双面汇总即可。

### 5.2 整袋流程

```text
上相机工位 presence 检测到袋
-> 上相机多光源采图 4 张
-> 上工位拨杆放行
-> 水样袋滑到下相机工位
-> 下相机 presence 检测到袋
-> 下相机多光源采图 4 张
-> 下工位拨杆放行
-> defect worker 汇总上/下两面检测结果
-> 末端拨杆分入 OK / NG 区
```

## 6. 相机、PLC、光源的时间戳对齐

这里的时间戳对齐不是为了做双相机同步，而是为了确认每张图是在正确光源状态下拍到的，并能在现场排查问题时还原动作时序。

### 6.1 推荐记录的时间戳

每一次采图 session 记录：

```text
capture_session_id
bag_id
station_id
camera_id
side_id
```

每一张光源图记录：

```text
light_id
light_name
plc_light_command_ts
light_settle_done_ts
camera_trigger_ts
camera_exposure_start_ts
camera_exposure_end_ts
image_received_ts
image_saved_ts
plc_motion_command_ts
```

如果相机 SDK 支持 chunk data，优先使用：

```text
camera_chunk_timestamp
camera_exposure_time
camera_gain
camera_frame_id
```

如果 PLC 支持读取内部时钟或动作反馈，则记录：

```text
plc_command_sent_ts
plc_output_on_ts
plc_output_off_ts
plc_ack_ts
```

### 6.2 时间源建议

优先级：

1. 相机硬件时间戳或 chunk timestamp。
2. PLC 反馈时间戳。
3. 工控机 `CLOCK_MONOTONIC_RAW` 时间戳。
4. 普通系统时间，仅用于人读日志。

如果相机和 PLC 没有统一时钟，需要在工控机侧记录：

```text
host_send_light_cmd_ts
host_trigger_camera_ts
host_image_rx_ts
host_plc_ack_ts
```

并通过固定延迟标定估算：

```text
PLC 输出延迟
光源响应延迟
相机触发延迟
曝光中心时间
```

### 6.3 采图时序建议

对每个光源 preset 执行：

```text
1. all_lights_off
2. PLC/light-controller 选择 light_id
3. 等待 light_settle_delay_us
4. 触发相机曝光
5. 等待曝光结束或接收 frame callback
6. 关闭当前光源
7. 保存图像和元数据
```

建议参数初值：

```text
light_settle_delay_us: 500 到 2000 us
exposure_us:
  背光: 50 到 300 us
  暗场: 100 到 800 us
  偏振漫射: 300 到 1500 us
gain: 尽量低，优先提高频闪亮度
```

如果 PLC 输出抖动较大，不建议让 PLC 直接承担微秒级光源时序。更稳的方式是：

```text
PLC 管机构动作
工控机/相机触发器/频闪控制器 管光源 burst 与相机曝光
PLC 只接收采图完成信号后执行拨杆动作
```

但如果现场必须由 PLC 控制光源切换，则需要：

- 每次光源切换后加入固定 settle delay。
- 记录 PLC 输出 ack。
- 将相机曝光触发放在光源稳定之后。
- 用示波器或高速采集卡标定输出到曝光的真实延迟。

建议每一面形成一个 `CaptureGroup`：

```json

  "bag_id": "bag_0001",
  "station_id": "upper_station",
  "camera_id": 1,
  "side_id": "front",
  "created_at": "...",
  "images": [
    {
      "light_id": "L1",
      "light_name": "backlight",
      "path": "captures/bag_0001/front/L1_backlight.png",
      "camera_frame_id": 101,
      "camera_exposure_start_ts": "...",
      "plc_light_command_ts": "...",
      "image_saved_ts": "..."
    },
    {
      "light_id": "L2",
      "light_name": "darkfield_a",
      "path": "captures/bag_0001/front/L2_darkfield_a.png"
    },
    {
```

defect_queue 中不再是单张图片，而是：

```text
DefectTask = CaptureGroup
```

```text
BagInspectionSession
  - upper CaptureGroup
  - lower CaptureGroup
  - upper defect results
  - lower defect results
  - final OK / NG decision
```

## 8. 检测模型与结果融合建议

### 8.1 按光源拆模型或拆分支

建议不要把所有光源图简单拼在一起直接训练一个模型。更稳的方式是：

```text
backlight_detector:
  输入 L1
  关注头发丝、异物、针孔、透光异常

darkfield_detector:
  输入 L2 + L3
  关注折痕、表面划痕、凸起边缘、贴附毛发

polarized_detector:
  输入 L4
  关注淡色污染、反光伪缺陷、浅色表面异常
```

最后做规则或学习型融合：

```text
任一关键缺陷命中 -> NG
弱缺陷多光源重复出现 -> 提高置信度
只在单一暗场方向出现且形态像正常折痕 -> 降低置信度或进入复检
```

### 8.2 坐标统一

同一相机同一工位下，多光源图像视角基本一致，但仍建议做：

- 镜头畸变校正。
- 固定 ROI 裁剪。
- 光源间图像尺寸一致。
- 如存在微小位移，标定单应性矩阵。

检测输出统一映射到同一物理坐标：

```text
x_mm, y_mm, width_mm, height_mm
```

这样上/下两面结果、不同光源结果都能合并。

## 9. 图像预处理建议

### 背光图 L1

建议：

- 平场校正，消除背光不均。
- Black-hat / Top-hat 增强细线和小黑点。
- 高通滤波增强头发丝。
- 对针孔可尝试局部阈值和连通域作为辅助特征。

### 暗场图 L2 / L3

建议：

- 保留高频边缘。
- 避免过强去噪抹掉细线。
- 可以计算 `max(L2, L3)` 增强任意方向亮线。
- 可以计算 `abs(L2 - L3)` 区分方向性纹理。

### 偏振漫射图 L4

建议：

- CLAHE 局部对比度增强。
- 低频背景校正。
- 与普通漫射图对比时可做反光抑制分析。

## 10. 与当前 C++ 后端流程的关系

当前 C++ 后端已经适合演进为下面的结构：

```text
station worker:
  presence 快速检测
  多光源 burst 采图
  PLC 拨杆放行
  CaptureGroup 入队

defect worker:
  读取 CaptureGroup
  按 light_id 调用不同检测器
  融合单面结果
  聚合上下两面结果
  输出 OK / NG

plc worker:
  控制相机工位上下拨杆
  控制末端分拣拨杆
  记录 ack / retry / timeout
```

需要注意：

- presence 检测是持续快速流程。
- 缺陷检测是异步慢流程。
- 多光源采图属于 station worker，必须在拨杆放行前完成。
- 末端分拣必须等待 defect worker 给出整袋最终结果。

## 11. 推荐验证实验

### 实验 1: 单光源对比

对同一批样本分别采集：

```text
普通正面光
背光
暗场 A
暗场 B
交叉偏振漫射
```

人工标注每种光源下缺陷可见性：

```text
0 = 不可见
1 = 勉强可见
2 = 清晰可见
```

统计不同缺陷类型的最佳光源。

### 实验 2: 多光源组合收益

比较：

```text
单 RGB 图
L1 背光
L1 + L2 + L3
L1 + L2 + L3 + L4
```

指标：

- 漏检率。
- 误检率。
- 单袋采图时间。
- 缺陷检测耗时。
- 末端分拣前是否能完成推理。

### 实验 3: 时间戳可靠性

对每张图检查：

```text
camera_trigger_ts - plc_light_command_ts
image_received_ts - camera_trigger_ts
plc_motion_command_ts - final_image_saved_ts
```

验证是否存在：

- 光源未稳定就曝光。
- 错光源图像。
- 拍照未完成就放行。
- 图像落盘与 bag_id 错配。

## 12. 推荐落地优先级

第一阶段：

- 增加 L1 背光、L2/L3 双向暗场。
- 每面 3 张缺陷图。
- 完成 CaptureGroup 数据结构和异步缺陷队列。

第二阶段：

- 增加 L4 交叉偏振漫射图。
- 引入光源级 detector 和结果融合。
- 增加平场校正和暗场方向融合。

第三阶段：

- 标定物理分辨率和 ROI。
- 接入相机 chunk timestamp。
- 用示波器标定 PLC 光源输出、相机曝光、频闪响应之间的真实延迟。

## 13. 高速多光源采图优化

多光源 burst 模块和三包 C++ 架构的详细设计见：

```text
项目附加文件/多光源Burst模块与三包C++架构设计.md
```

如果目标是让“换光源、拍图、再换光源、再拍图”尽量快，关键原则是：

```text
不要让 PLC 按帧串行控制每一次光源切换。
不要等待每张图存盘后再拍下一张。
不要在 burst 中做自动曝光、自动增益或软件握手。
```

推荐将多光源采图设计成硬件 burst：

```text
PLC 只发一次 start_capture_burst
-> 频闪控制器 / 相机 sequencer 执行预设光源序列
-> 相机连续采集多帧
-> PLC 收到 burst_done 后立即执行拨杆动作
-> 工控机异步收图并组包进入 defect_queue
```

### 13.1 最快的推荐硬件架构

```text
PLC:
  负责机构动作
  负责发 start_capture_burst
  负责接收 burst_done

相机:
  预先进入 armed / grabbing 状态
  使用硬触发或 sequencer 连续采集多帧
  每帧输出 frame_id / timestamp / sequencer_index

频闪控制器:
  预先配置 light sequence
  接收 trigger 后按帧点亮对应光源
  使用 TTL / opto input，而不是串口或 Modbus 临时改参数

工控机:
  不参与逐帧时序闭环
  只接收图像 callback
  按 capture_session_id + frame_index 组包
```

这样做的好处：

- PLC 不需要逐张图等待 ack。
- 光源切换由频闪控制器完成，延迟通常比 PLC 通信稳定。
- 相机提前处于采集状态，减少启动采集开销。
- 图像可以先进入内存 ring buffer，后续再落盘或送检测。

### 13.2 不推荐的慢流程

不要采用下面这种方式作为生产节拍：

```text
PLC/上位机打开 L1
-> 等待
-> 相机拍 L1
-> 保存 L1
-> PLC/上位机关闭 L1
-> PLC/上位机打开 L2
-> 等待
-> 相机拍 L2
-> 保存 L2
...
```

慢的原因：

- 每张图都有软件调用和通信往返。
- PLC 输出、相机触发、文件保存被串行化。
- 图像保存 I/O 会阻塞下一次曝光。
- 光源切换时间不可控，容易靠人为 delay 堆安全余量。

### 13.3 高速采图时序

推荐时序：

```text
t0:  presence 检测到袋体
t1:  生成 capture_session_id
t2:  PLC -> trigger controller: start_capture_burst
t3:  frame 0: light preset A on, camera exposure
t4:  frame 1: light preset B on, camera exposure
t5:  frame 2: light preset C on, camera exposure
t6:  optional frame 3: light preset D on, camera exposure
t7:  trigger controller -> PLC: burst_done
t8:  PLC 执行下拨杆 release、上拨杆 push、拨杆复位
t9:  工控机异步收到全部图像并放入 defect_queue
```

注意：

- 机构放行动作只需要等曝光完成，不需要等图像检测完成。
- 如果相机支持 exposure end 输出，可以用 exposure_end 或 frame_done 作为 burst_done 的依据。
- 工控机收到图像的时间可以晚于拨杆动作，不影响节拍。

### 13.4 推荐将光源帧数从 4 张优化为 3 张

为了速度，建议生产默认采用每面 3 张：

```text
Frame 0: L1 背光
Frame 1: L2 + L3 双向暗场同时点亮
Frame 2: L4 交叉偏振漫射
```

为什么可以把 L2 / L3 合并：

- 单独拍 L2 和 L3 的优势是能分析方向性。
- 但如果目标是快速检出，双向暗场同时点亮可以减少一帧。
- 对多数细线、折痕、附着异物来说，“是否可见”比“来自哪个方向”更重要。

这样每个水样袋：

```text
上面 3 张 + 下面 3 张 = 6 张检测图
```

相比 8 张图，采集时间和数据量下降约 25%。

如果节拍仍然紧张，可以采用极限快速模式：

```text
Frame 0: L1 背光
Frame 1: L2 + L3 双向暗场
```

即每面 2 张，每袋 4 张。代价是淡色污染、反光伪缺陷、浅色折痕区分能力下降，L4 可作为复检模式。

### 13.5 相机 ROI 和读出速度优化

多光源 burst 的速度经常不是曝光时间决定的，而是相机读出时间决定的。

优化建议：

- 使用相机硬件 ROI，而不是拍完整图后再裁剪。
- 只采集袋体有效区域，减少 sensor readout 行数。
- 使用 global shutter。
- 关闭自动曝光、自动增益、自动白平衡。
- 为每个光源 preset 固定 exposure / gain。
- 使用相机 sequencer 预加载每一帧的曝光参数。
- 图像先进入内存 ring buffer，不要在 burst 中同步写磁盘。

示例目标：

```text
presence ROI:
  低分辨率或缩小 ROI
  目标 5 到 15 ms 内完成判断

defect burst ROI:
  只覆盖检测有效区域
  3 帧 burst 目标 10 到 30 ms
  4 帧 burst 目标 15 到 45 ms
```

具体时间取决于相机分辨率、接口带宽、曝光、频闪亮度和 SDK 回调效率。

### 13.6 光源控制优化

建议：

- 使用多通道频闪控制器，提前配置每个通道的电流、脉宽、延迟。
- 使用 TTL 触发，不要在每帧通过串口/Modbus 设置光源参数。
- 使用高亮短脉冲，减少曝光时间和运动模糊。
- 光源切换时不要使用机械继电器，使用固态输出或专用 LED driver。
- 对 L2 + L3 双向暗场同时点亮时，要验证是否过曝或产生过多伪影。

如果必须由 PLC 控制光源：

- 不要让上位机逐帧发 Modbus 指令给 PLC。
- PLC 内部提前写好 burst function block。
- 上位机只发一次 start_burst。
- PLC 使用高速输出点按固定时序切光源和触发相机。
- 每个 output 的真实延迟需要用示波器标定。

### 13.7 图像接收与缓存优化

工控机侧建议：

```text
Camera callback
-> lock-free / low-lock ring buffer
-> attach metadata
-> CaptureGroup assembler
-> defect_queue
```

不要在相机 callback 里做：

- 缺陷推理。
- 磁盘保存大图。
- 图像格式复杂转换。
- 日志大量打印。

callback 里只做：

```text
拷贝或引用 buffer
记录 frame_id / timestamp / light_index
快速入队
```

图像保存可以放到后台 storage worker。

### 13.8 时间戳在高速方案中的作用

高速方案中，时间戳不是用来“控制每帧什么时候拍”，而是用于验证和追溯：

```text
frame_index 是否对应正确 light_id
光源切换是否早于 exposure_start
burst_done 是否晚于最后一帧 exposure_end
PLC 拨杆动作是否晚于 burst_done
```

真正保证高速和稳定的是：

- 预配置光源序列。
- 硬件触发。
- 相机 sequencer。
- 频闪控制器。
- ring buffer 异步收图。

时间戳负责证明这套硬件时序确实按预期执行。

### 13.9 推荐生产节拍方案

综合速度和检出率，推荐默认生产方案：

```text
presence:
  L0 常亮或快速触发
  只做 bag_present 判断

defect burst per side:
  Frame 0: L1 背光
  Frame 1: L2 + L3 双向暗场
  Frame 2: L4 交叉偏振漫射

per bag:
  上面 3 张 + 下面 3 张 = 6 张检测图
```

如果后续实验发现 L4 收益不明显，可降级为：

```text
上面 2 张 + 下面 2 张 = 4 张检测图
```

如果后续实验发现暗场方向性非常重要，再升级为：

```text
上面 4 张 + 下面 4 张 = 8 张检测图
```

也就是：

```text
速度优先: 2 张/面
生产平衡: 3 张/面
质量优先: 4 张/面
```

## 14. 最终推荐配置

最终推荐质量优先配置：

```text
presence:
  L0 常亮或快速触发

defect burst per side:
  L1 背光
  L2 暗场 A
  L3 暗场 B
  L4 交叉偏振漫射

per bag:
  上面 4 张 + 下面 4 张 = 8 张检测图
```

如果节拍压力过大，先采用：

```text
上面 3 张 + 下面 3 张 = 6 张检测图
```

并保留 L4 作为复检或后续升级项。
