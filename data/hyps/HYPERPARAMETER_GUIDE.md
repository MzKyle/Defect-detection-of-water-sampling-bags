## 📖 详细参数解释

### 🔵 学习率相关参数

#### `lr0` - 初始学习率
```yaml
lr0: 0.01  # SGD=1E-2, Adam=1E-3
```
**含义**：训练开始时的学习率（步长）
- **SGD 优化器**：通常 0.01（较大）
- **Adam 优化器**：通常 0.001（较小，Adam 自适应）
- **预训练模型**：可用更小的值（0.0001-0.001）

**调整建议**：
| 场景 | 建议值 | 原因 |
|------|------|------|
| 从零训练 (scratch) | 0.01 | 需要充分探索 |
| 迁移学习 (pretrained) | 0.001-0.0005 | 权重已经优化 |
| 数据很少 | 0.001 | 防止过拟合 |
| 大模型 (L/X) | 0.005 | 大模型容易振荡 |

#### `lrf` - 最终学习率系数
```yaml
lrf: 0.01  # 最终 lr = lr0 × lrf
```
**含义**：学习率衰减的目标系数
- **计算**：final_lr = lr0 × lrf = 0.01 × 0.01 = 0.0001
- **衰减方式**：线性衰减从 lr0 到 final_lr

**调整建议**：
| lrf 值 | 衰减幅度 | 用途 |
|------|---------|------|
| 0.01 | **强衰减** | 需要精细调优的阶段 |
| 0.1 | **温和衰减** | 保持一定学习率到最后 |
| 0.2 | **缓慢衰减** | 长期保持学习能力 |

#### `warmup_epochs` - 预热轮数
```yaml
warmup_epochs: 3.0
```
**含义**：学习率从 0 线性增长到 lr0 的轮数

**工作流程**：
```
第 1 epoch：lr = 0.003
第 2 epoch：lr = 0.006
第 3 epoch：lr = 0.009
第 4 epoch：lr = 0.01 (开始衰减)
```

**调整建议**：
- **小模型 (S/N)**：warmup_epochs = 3
- **大模型 (L/X)**：warmup_epochs = 5-10
- **数据很少**：warmup_epochs = 5-10 （防止初期振荡）

#### `warmup_momentum` - 预热初始动量
```yaml
warmup_momentum: 0.8
```
**含义**：预热期间的初始动量值
- **预热期**：从 warmup_momentum 线性增长到 momentum
- **示例**：从 0.8 → 0.937（9 epochs 内）

### 🟢 优化器相关参数

#### `momentum` - SGD 动量
```yaml
momentum: 0.937
```
**含义**：SGD 优化器的动量系数（加速度）

**物理意义**：
```
当前更新 = momentum × 上次更新 + (1 - momentum) × 当前梯度
```

**推荐值**：
- **0.9**：较弱的记忆，快速响应新梯度
- **0.937**：平衡记忆和反应性（**YOLOv5 默认**）
- **0.99**：强记忆，平滑更新轨迹

#### `weight_decay` - 权重衰减（L2 正则化）
```yaml
weight_decay: 0.0005
```
**含义**：每步训练时对权重的惩罚系数

**效果**：
```
loss_total = loss_original + weight_decay × ||weights||²
```

**调整建议**：
| 值 | 效果 | 用途 |
|----|------|------|
| 0 | 无正则化 | 过拟合严重时 |
| 0.0001-0.0005 | **标准** | 大多数情况 |
| 0.001 | 强正则化 | 小数据集 |
| 0.005+ | 极强正则化 | 严重过拟合 |

#### `warmup_bias_lr` - 预热期 Bias 学习率
```yaml
warmup_bias_lr: 0.1
```
**含义**：预热期间的 bias 项学习率

**原理**：Bias 项学习率独立设置，通常保持较小值
- **标准值**：0.1（相对于 lr0）
- **无特殊需求**：无需调整

### 🟡 损失函数权重

#### 基础损失权重

```yaml
box: 0.05        # 边界框损失（L1/GIoU）
cls: 0.3         # 分类损失（BCELoss）
obj: 0.7         # 目标置信度损失（BCELoss）
```

**含义**：
- **总损失** = box_loss × box + cls_loss × cls + obj_loss × obj

**调整原则**：

| 参数 | 增大的效果 | 减小的效果 | 何时调整 |
|------|-----------|-----------|---------|
| **box** | 边界框更精准 | 定位更松散 | 定位精度要求高 |
| **cls** | 分类更准确 | 背景误检减少 | 需要精准分类 |
| **obj** | 减少漏检 | 减少误检 | **通常保持默认** |

**实际调优示例**：
```yaml
# ❌ 定位精度不足
box: 0.05, cls: 0.3, obj: 0.7

# ✅ 改进：加强 box 权重
box: 0.1, cls: 0.3, obj: 0.7

# ❌ 背景误检太多
box: 0.05, cls: 0.3, obj: 0.7

# ✅ 改进：增大 obj 权重抑制背景
box: 0.05, cls: 0.3, obj: 1.0
```

#### BCE 权重参数

```yaml
cls_pw: 1.0      # 分类损失正样本权重
obj_pw: 1.0      # 目标损失正样本权重
```

**含义**：二分类交叉熵中正样本的加权系数

**应用**：
- **cls_pw=1.0**：正类和负类同等重要
- **cls_pw=2.0**：正类（实际目标）重要性加倍
- **obj_pw>1.0**：加强对真实目标的检测

**调整建议**：
| 问题 | 调整 | 效果 |
|------|------|------|
| 漏检率高 | cls_pw↑, obj_pw↑ | 更容易检测出目标 |
| 误检率高 | cls_pw↓, obj_pw↓ | 更严格的检测 |

#### IoU 和锚框阈值

```yaml
iou_t: 0.20          # IoU 训练阈值
anchor_t: 4.0        # 锚框倍数阈值
```

**iou_t 含义**：
- 预测框与真值框的 IoU > iou_t 时，视为正样本
- **iou_t = 0.20**：较宽松（大多数预测都算正样本）
- **iou_t = 0.50**：严格（只有高度重叠才算正样本）

**anchor_t 含义**：
- 防止锚框与真值框尺寸偏差过大
- **anchor_t = 4.0**：允许 4 倍尺寸差异
- 计算：max(anchors/gt_box, gt_box/anchors) < anchor_t

**调整建议**：
```yaml
# 小目标多的数据集
iou_t: 0.25        # 提高正样本比例
anchor_t: 5.0      # 允许更大偏差

# 大目标多的数据集
iou_t: 0.15        # 降低正样本比例
anchor_t: 3.0      # 要求更精确
```

### 🔴 数据增强参数

#### 颜色空间增强（HSV）

```yaml
hsv_h: 0.015       # 色调增强 (0.015 = ±1.5%)
hsv_s: 0.7         # 饱和度增强 (70% 变化)
hsv_v: 0.4         # 亮度增强 (40% 变化)
```

**工作原理**：
```python
# 随机在范围内增强
h = h + random(-hsv_h, hsv_h)
s = s × random(1-hsv_s, 1+hsv_s)
v = v × random(1-hsv_v, 1+hsv_v)
```

**增强效果**：
| 参数 | 低值 | 高值 | 调整条件 |
|------|------|------|---------|
| hsv_h | 自然色调 | 极端色调 | 数据有多种光照 |
| hsv_s | 灰白影像 | 饱和鲜艳 | 对比度变化大 |
| hsv_v | 暗图像 | 亮图像 | 亮度变化大 |

**建议值**：
```yaml
# 自然光照环境
hsv_h: 0.015, hsv_s: 0.7, hsv_v: 0.4

# 极端光照条件
hsv_h: 0.03,  hsv_s: 0.9, hsv_v: 0.6

# 光照已均衡（合成数据）
hsv_h: 0.01,  hsv_s: 0.3, hsv_v: 0.2
```

#### 几何变换

```yaml
degrees: 0.0       # 旋转角度 (±0°)
translate: 0.1     # 平移比例 (±10% 图像大小)
scale: 0.9         # 缩放因子 (0.9-1.1 倍)
shear: 0.0         # 剪切变换 (±0°)
perspective: 0.0   # 透视变换 (0-0.001)
```

**每个参数的含义**：

| 参数 | 范围 | 示例 | 适用场景 |
|------|------|------|---------|
| **degrees** | ±度数 | `degrees: 10` = ±10° | 目标可旋转 |
| **translate** | ±比例 | `translate: 0.2` = ±20% | 目标位置变化 |
| **scale** | ±因子 | `scale: 0.5` = 0.5-1.5 倍 | 距离变化 |
| **shear** | ±度数 | `shear: 5` = ±5° | 倾斜视角 |
| **perspective** | 0-0.001 | `perspective: 0.0005` | 斜视图 |

**调整建议**：
```yaml
# 场景 1：固定角度、固定位置的工业检查
degrees: 0.0
translate: 0.05
scale: 0.2
shear: 0.0
perspective: 0.0

# 场景 2：自然场景、多角度拍摄
degrees: 15.0
translate: 0.1
scale: 0.5
shear: 2.0
perspective: 0.0005

# 场景 3：安防摄像头（高度变化）
degrees: 5.0
translate: 0.2
scale: 0.8
shear: 0.0
perspective: 0.001
```

#### 翻转变换

```yaml
flipud: 0.0        # 上下翻转概率 (0% = 从不翻转)
fliplr: 0.5        # 左右翻转概率 (50%)
```

**含义**：
- **flipud = 0.5**：50% 概率上下翻转图像
- **fliplr = 0.5**：50% 概率左右翻转图像

**调整建议**：
```yaml
# 目标方向无意义（如行人）
flipud: 0.5        # 可以翻转
fliplr: 0.5

# 目标有特定方向（如文字、箭头）
flipud: 0.0        # 禁止翻转
fliplr: 0.0

# 目标对称性强（如车牌）
flipud: 0.0
fliplr: 0.5        # 仅左右翻转
```

#### 高级增强

```yaml
mosaic: 1.0        # Mosaic 增强概率 (YOLOv5 特色)
mixup: 0.1         # MixUp 增强概率
copy_paste: 0.0    # CopyPaste 增强概率 (需标注实例)
```

**Mosaic 增强** (概率 = 1.0 = 总是启用)
```
功能：将 4 张图拼接成 1 张，增加单张图像的目标数量
优点：加快收敛，学习小目标
```

**MixUp 增强**
```
功能：线性混合两张图：img = α×img1 + (1-α)×img2
优点：增加数据多样性，提高泛化能力
```

**CopyPaste 增强**
```
功能：从一张图复制目标贴到另一张图
仅用于：有像素级标注时（mask）
```

**调整建议**：
```yaml
# 默认设置（适合大多数情况）
mosaic: 1.0
mixup: 0.1
copy_paste: 0.0

# 数据不足时
mosaic: 1.0
mixup: 0.3         # 增加 mixup
copy_paste: 0.0

# 数据充足、已有实例分割标注
mosaic: 0.8        # 降低 mosaic，避免过度增强
mixup: 0.05
copy_paste: 0.2
```

### 🟣 特殊参数

#### `fl_gamma` - Focal Loss 参数
```yaml
fl_gamma: 0.0      # 0 = 不使用, > 0 = 启用 Focal Loss
```

**含义**：处理类别不平衡问题

**工作原理**：
```
Focal Loss = -(1 - p_t)^γ × log(p_t)
```

**调整建议**：
| 值 | 用途 |
|----|------|
| 0.0 | **标准** BCE Loss |
| 1.5 | 轻度不平衡 |
| 2.0 | 中度不平衡（IoU 较低的样本加权） |

**使用场景**：
```yaml
# 正负样本严重不平衡
fl_gamma: 2.0

# 样本均衡
fl_gamma: 0.0
```

#### `anchors` - 锚框数量
```yaml
anchors: 3         # 每个输出层的锚框数（可选）
```

**说明**：
- 注释掉时：使用默认锚框配置
- 设置值时：自动生成指定数量的锚框

## 🎯 参数调优流程

### 📊 诊断问题：数据指标不好

```mermaid
问题诊断树
├─ 漏检率高（Recall 低）
│  ├─ 增大 obj 权重：obj: 0.5 → 1.0
│  ├─ 降低 IoU 阈值：iou_t: 0.2 → 0.15
│  └─ 增加增强强度：hsv_h, hsv_s, hsv_v ↑
│
├─ 误检率高（Precision 低）
│  ├─ 降低 obj 权重：obj: 1.0 → 0.5
│  ├─ 增加 IoU 阈值：iou_t: 0.2 → 0.25
│  └─ 减少增强强度：mosaic ↓, mixup ↓
│
├─ 定位不准确（Box Loss 高）
│  ├─ 增加 box 权重：box: 0.05 → 0.1
│  ├─ 降低学习率：lr0: 0.01 → 0.005
│  └─ 多加预热轮数：warmup_epochs: 3 → 5
│
└─ 过拟合（Val Loss > Train Loss）
   ├─ 增加增强：hsv_*, mosaic, mixup ↑
   ├─ 增加正则化：weight_decay ↑
   └─ 降低学习率：lr0 ↓
```

### 📋 调优检查清单

```
□ 数据质量检查
  □ 标注是否正确
  □ 是否有类不平衡

□ 参数初始化
  □ 选择合适的 hyp 文件（low/med/high）
  □ 设置正确的学习率（lr0）

□ 训练监控
  □ 检查训练曲线是否平稳
  □ Box/Cls/Obj 损失是否均衡

□ 逐步调优
  □ 一次只改一个参数
  □ 观察 3-5 个 epoch 的变化
  □ 记录每个改动的效果

□ 最终验证
  □ 在验证集评估
  □ 检查各类别性能
  □ 保存最优权重
```

## 📚 推荐阅读顺序

1. **快速开始**：选择合适的 hyp 文件 (low/med/high)
2. **基础理解**：学习学习率和优化器参数
3. **数据增强**：调整 HSV 和几何变换
4. **损失调优**：微调各损失函数权重
5. **高级技巧**：使用 Focal Loss 和其他特殊参数

## ⚠️ 常见错误

| ❌ 错误做法 | ✅ 正确做法 | 原因 |
|-----------|-----------|------|
| 一次改多个参数 | 一次只改一个 | 无法判断哪个参数有效 |
| 只看 train loss | 同时看 val loss | 防止过拟合 |
| 用错的 hyp 文件 | 根据数据量选择 | 参数预设针对特定场景 |
| 不调整 lr | 根据不同任务改 lr | 学习率是最重要参数 |
